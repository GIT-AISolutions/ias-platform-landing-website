OpenCodie Platform Guide

OpenCodie is an online development platform that helps you build, manage, and deploy projects from one place — without constantly switching between separate tools.

Updated May 5, 2026 ~25 min read 21 sections

What is OpenCodie?

OpenCodie helps you build and manage websites, web apps, and software projects faster. Normally, you often need multiple tools: a code editor, terminal, hosting platform, database panel, AI chat, and deployment tool. OpenCodie brings these parts together in one platform.

FeatureWhat you can do
ProjectsOrganize everything related to one website, app, or client
WorkspacesCode online in a safe development environment
EditorEdit code directly in the browser
TerminalRun commands without installing everything locally
PreviewView your website or app while building
AI ChatGet help with code, bugs, explanations, and improvements
DeploymentsPublish your project to production
DatabasesView and manage data through a safe viewer
SecretsStore API keys and passwords securely
TeamsCollaborate with multiple users

Who is OpenCodie for?

OpenCodie is built for people and teams who want to build software faster without managing many separate tools.

  • Developers who want to work on projects online
  • Agencies managing multiple client projects
  • Teams working together on websites or apps
  • Makers who want to quickly build and launch ideas
  • Companies that want development, hosting, and AI help in one place

You do not always need to install everything on your own computer. Most work can be done directly from the browser.

Getting started

Create an account

  1. Go to the OpenCodie portal and click Sign up.
  2. Enter your email address and password, or use a supported OAuth provider.
  3. Confirm your email address through the verification email.

Choose or create an organization

After logging in, you can select an existing organization or create a new one. An organization groups your projects, workspaces, team members, billing, and usage in one place.

Create your first project

  1. Go to Projects in the navigation.
  2. Click Create project.
  3. Enter a clear project name.
  4. Confirm the project creation.
  5. The project will appear in your project overview.

Start your first workspace

  1. Open your project.
  2. Click New workspace or + Workspace.
  3. Choose a name and configuration.
  4. Click Create workspace.
  5. Wait until the status changes to Running.
  6. Click Open workspace to start working.

A workspace can take a short moment to start because OpenCodie prepares the development environment for your project.

How does the platform work?

OpenCodie uses a simple hierarchical structure:

Organization
└── Project
    ├── Workspace
    ├── Deployment
    ├── Database
    └── Secrets

A project is the main container for everything related to a website, app, product, or client. Inside a project, you can have multiple parts.

Project: SaaS Platform
├── Workspace: Frontend app
├── Workspace: Backend API
├── Deployment: Production frontend
├── Deployment: Production backend
├── Database: PostgreSQL
└── Secrets: Stripe, OpenAI, and database keys

Projects

A project is where you organize everything. Use one project per client, website, app, or product.

When should you create a new project?

SituationBest choice
New client websiteNew project
Extra landing page for the same clientNew workspace inside the existing project
Separate SaaS appNew project
Backend and frontend of the same appOne project with multiple workspaces

Naming tip: Use clear project names like JT Fitness Website, OpenCodie Platform, or Client Name — New Website to keep the platform organized when you have many projects.

Workspaces

A workspace is your online development environment — where you write code, run commands, and preview what you are building. You can think of a workspace as an online version of your local development environment.

What is inside a workspace?

PartDescription
EditorEdit code in the browser
TerminalRun commands like npm install, npm run dev, or git status
PreviewView your website or app directly
AI ChatGet help with code, bugs, and improvements
FilesBrowse and edit your project files
UploadsUpload files as context for the AI chat

Workspace status flow

Stopped → Starting → Active
Active  → Stopping → Stopped
Active  → Restarting → Active

Stop a workspace when you are done, so you do not use unnecessary resources.

Multiple workspaces

Project: SaaS Platform
├── Workspace: Frontend
├── Workspace: Backend API
└── Workspace: Marketing Website

Editor, terminal, and preview

OpenCodie combines the editor, terminal, and preview in one screen.

Editor

The editor supports HTML, CSS, JavaScript, TypeScript, React, Next.js, Python, FastAPI, and configuration files.

Terminal

The terminal lets you run commands inside your workspace. Common commands:

npm install
npm run dev
npm run build
git status
git add .
git commit -m "Update project"
git push

AI Chat and code suggestions

Each workspace includes an AI assistant that can help with your project.

  • Find bugs and explain code
  • Build new components
  • Analyze error messages
  • Create a step-by-step plan
  • Suggest a code patch

Example prompts

Can you check why my build is failing?
Make this homepage more mobile-friendly without changing the style.
Explain what this API route does.
Create a safe patch to fix this issue.

Deployments

A deployment is the live version of your project. OpenCodie supports multiple stacks and in most cases auto-detects your project type.

Supported stacks

  • Static HTML
  • Vite / React (Vite)
  • Next.js
  • Node + Express
  • Python FastAPI / Flask / Django
  • Astro, SvelteKit, Nuxt, Storybook
  • Custom (with your own build configuration)

Databases

Databases let you store data for your project — users, products, settings, orders, or other application data.

Supported engines

  • PostgreSQL, MySQL, MariaDB
  • MongoDB
  • Redis, Valkey / KeyDB, Dragonfly
  • ClickHouse

Safe database viewer

The database viewer is meant for safely viewing data with built-in restrictions:

  • Read-only queries only
  • Row limits and query timeouts
  • Only allowed schemas are visible
  • Sensitive columns can be hidden or masked

Use the database viewer to quickly check whether data is being stored correctly. It is not meant to replace full database management tools for complex operations.

Secrets

Secrets are sensitive values such as API keys, tokens, passwords, and database URLs.

OPENAI_API_KEY
STRIPE_SECRET_KEY
DATABASE_URL
SMTP_PASSWORD
GITHUB_TOKEN

Scope

ScopeUsage
ProjectShared with all workspaces and deployments inside the project
WorkspaceOnly available in one workspace
DeploymentOnly available in one live deployment

Security: Never place real secrets in chat messages, screenshots, or public documentation. Use the OpenCodie secrets panel to store them securely.

Logs and status messages

Logs help you understand what is happening in a workspace, job, or deployment. You can use logs to see whether an app starts correctly, why a build fails, which error a deployment gives, and whether commands ran successfully.

What to look for

  • The first error is usually the most important one
  • Later errors are often caused by the first failure
  • OpenCodie streams logs live in real-time
  • Sensitive values are filtered from logs automatically

Teams and organizations

Organizations let you collaborate with multiple users. An organization can contain multiple projects. Inside an organization, you can invite members and assign roles.

RoleWhat someone can do
OwnerFull control, including billing and deleting the organization
AdminManage members and organization settings
MemberView and work on projects, depending on permissions

Only give admin access to people who actually need to manage organization settings or members.

Billing, limits and usage

OpenCodie uses subscriptions and usage tracking. Depending on your plan, you get access to certain resources. Usage is measured based on workspace hours, deployments, databases, and extra resource usage.

How to prevent unexpected costs

  • Stop workspaces you are not using
  • Check the billing and usage overview regularly
  • Keep an eye on active deployments and builds
  • Enable extra usage only when you understand the cost impact

A running workspace keeps using resources while it is active. Stop workspaces when you are done working to keep usage and costs under control.

Why something works in workspace but not live

  • A secret exists in the workspace but not in the deployment
  • An environment variable has a different name or value in production
  • A dependency is missing from package.json
  • The build command differs from development
  • The app expects a local file or service that does not exist in production

Troubleshooting

Build fails

  1. Open the deployment logs.
  2. Look for the first real error message.
  3. Check for missing dependencies, syntax errors, or wrong environment variables.
  4. Fix the issue in your workspace, commit, and deploy again.

Preview does not work

  1. Check whether the workspace status is Running.
  2. Make sure your app is actually started in the terminal.
  3. Confirm that the app is listening on the expected port.
  4. Restart the preview if the app configuration changed.

Domain or DNS does not work

  1. Check whether the domain is correctly added to deployment settings.
  2. Confirm that DNS records are configured at your domain provider.
  3. Remember that DNS changes can take time to propagate.
  4. Check whether the SSL certificate is active.

The first error in the logs is usually the most important one. Later errors are often caused by the first failure.

Why AI actions must be reviewed

AI can help with code, commands, and file changes, but it should not replace human review. Always review AI-generated changes before applying them, especially when the action can delete files, change database logic, modify authentication, or deploy to production.

Never place real secrets in chat messages, screenshots, or public repositories.

Frequently asked questions

Do I need to install anything locally?

Not always. You can do a lot from the browser: edit code, run commands, view previews, and use AI. For some advanced workflows, GitHub CLI or local tooling can still be useful.

What is the difference between a project and a workspace?

A project is the main place where everything is organized. A workspace is where you actually write code and test changes.

Project: Webshop
Workspace: Frontend build
Workspace: Backend build
Deployment: Live webshop
Database: Product data

Can AI directly edit my code?

Yes, but usually through a suggested patch. You can review the diff first and approve it before it is applied. This helps you stay in control of your code.

Can I use my own AI tools?

Yes. You can use your own AI tools through the terminal when they provide a CLI — Claude, Gemini, Codex, OpenAI CLI, or other supported AI command-line tools.

Can I connect my own domain?

Yes. Deployments can be connected to a custom domain. You need to configure DNS records at your domain provider.

When should I stop a workspace?

Stop a workspace when you are done working or when you do not need it for a while. This prevents unnecessary resource usage.

Best practices

Keep projects organized

Use one project per client, product, or large app. Create multiple workspaces inside the same project when the parts belong together.

Follow a fixed workflow

1
Open project and start workspace
2
Edit code in the editor
3
Test in preview
4
Ask AI for review or improvements
5
Run build and tests
6
Commit and deploy
7
Check logs after deployment

Use git before large changes

Before making big changes, create a commit or a new branch. This makes it easier to compare, review, or roll back changes.

Use AI as an assistant, not a replacement

  • Check whether the change makes sense
  • Verify the app still builds and the preview works
  • Ensure no secrets or sensitive data were added
  • Confirm the change matches the goal of the project

Summary: Use projects to stay organized, workspaces to build, deployments to go live, databases to manage data, secrets for safe configuration, and AI Chat to work faster.
